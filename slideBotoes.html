O que é Reúso de Software?

É o processo de criar software a partir de partes já existentes, em vez de começar do zero.

"Não reinventar a roda."

Componentização

Separar o sistema em partes reutilizáveis e independentes.

É possível pensar em peças de LEGO que se encaixam em diversas outras.

De forma mais técnica, um componente de login usado em várias páginas diferentes.

Componentização

Vantagens

  • Organização
  • Reuso
  • Manutenção facilitada

Desvantagens

  • Complexidade inicial
  • Sobrecarga de estrutura

O Ciclo de Vida de um Componente

1. Qualificar

Identificar candidatos a componentes reutilizáveis.

2. Generalizar

Adaptar o código para ser independente do contexto.

3. Compor

Montar a aplicação final usando os componentes.

A "Fábrica" de Software

Uma SPL cria uma plataforma base (ativos de núcleo) e a partir dela, gera uma família de produtos com variações específicas.

Por que usar uma SPL?

  • Velocidade: Lançar novos produtos se torna muito mais rápido.
  • Confiabilidade: O núcleo é amplamente testado e validado.
  • Padronização: Garante consistência e facilita o trabalho da equipe.

O Preço a Pagar

  • Planejamento: Exige um altíssimo investimento e planejamento inicial.
  • Risco Centralizado: Um erro no núcleo afeta TODOS os produtos.
  • Evolução Lenta: Mudar a plataforma base é um processo complexo.

Produtos COTS

Produtos COTS são soluções de software ou hardware prontas para uso, compradas no mercado.

Oferecem uma alternativa eficiente e de custo-benefício para construir algo do zero.

COTS no nosso dia a dia

Suítes de Produtividade

Microsoft 365, Google Workspace

Sistemas de Gestão (ERP/CRM)

SAP, Salesforce, Oracle

Bancos de Dados

MySQL, PostgreSQL, SQL Server

Ferramentas de Comunicação

Slack, Microsoft Teams, Zoom

COTS: Configurar vs. Integrar

COTS Configurável

Foco: Ajustar o produto para se adequar aos processos de negócio.

Como: Através de painéis administrativos, menus e opções pré-definidas, sem alterar o código.

Exemplo: Habilitar ou desabilitar módulos em um sistema ERP.

COTS Integrado

Foco: Conectar o COTS com outros sistemas para que funcionem juntos.

Como: Usando APIs, webhooks e conectores para permitir a troca de dados e automação.

Exemplo: Ligar um e-commerce (COTS) a um sistema de pagamento (outro COTS).

O Dilema: Construir vs. Comprar

Construir

+ Controle total e customização.

+ Vantagem competitiva única.

- Alto custo e longo prazo.

- Exige equipe e maior risco.

Comprar (COTS)

+ Implementação rápida e custo menor.

+ Suporte e manutenção garantidos.

- Menos flexível, exige adaptação.

- Dependência do fornecedor.

Geradores de Programas

Um Gerador acelera o desenvolvimento e padroniza a qualidade do código.

Pense em uma máquina de café: você não precisa saber moer os grãos ou aquecer a água. Você apenas escolhe o que quer (um expresso) e a máquina o gera para você.

Subindo na Escada da Abstração

GERADOR

A ferramenta que transforma a especificação em código funcional. É o "motor" da automação.

MODELO (UML, DSL)

Uma descrição de alto nível do que o software deve fazer, usando linguagens visuais ou específicas de domínio.

CÓDIGO (JAVA, PYTHON)

O produto final: código-fonte pronto para ser compilado e executado, totalmente gerado a partir do modelo.

Menos preocupação com o "como", mais foco no "o quê".

Conclusões
